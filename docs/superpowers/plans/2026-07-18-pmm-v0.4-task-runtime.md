# pmm v0.4 Task Runtime Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:test-driven-development` and `superpowers:writing-skills`. Execute inline because the runtime scripts, templates, and documentation share one contract and overlapping files.

**Goal:** Upgrade `pmm` from a free-form single-task convention to a backward-compatible, concurrency-aware task runtime with structured state, evidence freshness, deterministic recovery, and safe migration.

**Architecture:** Keep Markdown as the human-readable source of truth, add a small YAML frontmatter contract for machine state, and centralize dependency-free parsing/hashing in a shared shell library. Keep `active-task.md` as one primary task; concurrent feature conversations use branch-isolated work-item files, and lifecycle commands refuse ambiguous or same-branch claims.

**Tech Stack:** POSIX-oriented Bash, Git, `rg`, Markdown/YAML frontmatter, existing public-safety and sync scripts.
